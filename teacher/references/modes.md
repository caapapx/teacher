# Seven Teaching Modes

Read the section for the mode selected by routing. Simplify output structure when time is short or the problem is simple.

## Mode A: Guided Decomposition

**For:** Step-by-step solving — math, physics, case analysis, logical reasoning, algorithms, data structures, LeetCode-style problems.

Looking at the answer directly causes an "illusion of understanding." Break down the way the human brain actually thinks.

**Output structure (simplify based on complexity):**

1. **Initial intuition** — First reaction? Similar problems? Key differences? (comparison table)
2. **Concrete thinking** — Work through 1–2 small examples by hand; mark the "aha!" moment
3. **Key element checklist** — Element | Type | Meaning | When it changes | Why it matters  
   (CS: Variable | Type | Meaning | Update timing | Why needed)
4. **Reasoning chain** — Options at each step; highlight correct path and pitfalls  
   (CS: state transitions with positive/negative/boundary cases)
5. **Hand-trace verification** — Step-by-step table with state snapshots  
   (CS: simulate code execution)
6. **Method-to-thinking mapping** — Framework + "thinking → method" table  
   (CS: code skeleton + "thinking → code" mapping)
7. **Common mistakes and memory anchors** — 2–3 traps + short mnemonic
8. **Self-check list** — 3–5 verifiable criteria

## Mode B: Socratic + Advanced Recall

**For:** Exam / interview / oral defense prep, concept deepening, verifying depth.

Default: don't give a complete answer first — guide through a question chain. Only give a compressed answer if the user asks for a demo, is very pressed for time, or fails two rounds.

**Flow:**

1. **Baseline probe** — What does the user already know? Where are they confused?
2. **Question chain** — Foundational → applied → analytical → deep reasoning
3. **Guide, don't correct** — Wrong answers → ask a question that exposes the contradiction
4. **Summarize** — 3–5 points the user discovered themselves
5. **Gap check** — Weak spots + next steps

For exam/interview: How would an examiner follow up? How many layers deep?

---

## Mode C: Mental Model + Dual Coding

**For:** Complex or abstract concepts — theory, mechanisms, systems (including GMP, GC, distributed consistency, Transformer attention, etc.)

**Flow:**

1. **Ask about existing frameworks** — Determines which analogies fit
2. **Build a model diagram** — ASCII / tables / flowcharts: principle → rules → examples
3. **Dual coding** — Text + visual for each key point
4. **Connection test** — 1–2 test questions; expand to 3–5 only if user wants deep practice
5. **Transfer exercise** — What else can this model explain?

---

## Mode D: Simplification + Analogy

**For:** Introductory concepts, jargon intimidation, intuition-first learning

**Flow:**

1. **12-year-old version** — Jargon-free "what it fundamentally is"
2. **Everyday analogy** — State where it matches and where it differs
3. **Return to technical object** — Map analogy to real terms and boundaries
4. **Minimal example** — One concrete example (code snippet for CS)
5. **Counter-analogy** — Where the analogy breaks down
6. **Retelling task** — User explains back or invents their own analogy

---

## Mode E: Deep Elaboration

**For:** Principles, mechanism analysis, "why is it this way" (including source-level CS questions)

**Flow:** After user states a fact, ask in sequence:
1. Why is it true?
2. How does it work?
3. What would break it?
4. What if you changed X?

Summarize final understanding and mark depth leaps.

---

## Mode F: Interleaved Practice + Generative Learning

**For:** Systematic study, building a complete knowledge base

**Flow:**

1. Baseline assessment
2. 30–45 minute study plan — mix related concepts
3. Interleaved problems — default 3–5; 8–12 only in long sessions
4. Generative tasks — summary, analogy, new example, or 3 flashcards (default 1–2)
5. Error review loop

---

## Mode G: Metacognitive Strategy

**For:** Low efficiency, not knowing how to study; also embed proactively in other modes

**Before learning:** Verifiable goal in one sentence; identify weakest uncertainty  
**During (every 3–5 exchanges):** What feels solid vs shaky; reframe if stuck  
**After:** One-sentence retrieval; Feynman-style "explain to someone tomorrow"

**Active trigger:** When user says can't focus / no efficiency — diagnose cause, recommend method, adjust plan.
